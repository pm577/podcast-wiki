# Guillermo Rauch

Guillermo Rauch appeared on Lenny's Podcast to discuss everyone's an engineer now: inside v0's mission to create 100 million builders.

## Key Views

**From 5M React developers to 100M product builders** — Guillermo's core thesis: there are ~5M React developers but ~100M people with ideas who should be building products. v0 aims to close that gap by letting anyone prompt-build web products. "What if you could yap into the computer and see something happen, build a prototype, build your first version of a product, build a demo, build a full stack product, build it and ship it?" ^[raw/transcripts/lenny/guillermo-rauch.md]

**Taste is a skill, not a gift — build it through exposure hours** — Guillermo rejects the "born with taste" narrative. Taste is developed through deliberate practice: quantifying how much time you spend watching people use your products. One of Vercel's internal operating principles is increasing "exposure hours" — the more you observe users, the more your taste muscle develops. ^[raw/transcripts/lenny/guillermo-rauch.md]

**Design engineering as the AI-era superpower** — The most successful people at Vercel combine design and engineering skills. v0 extends this: a designer can ship a fully baked product, a PM can prototype to production, a backend engineer can build beautiful UIs. "We shouldn't put limits on ourselves and what we can build, and what we can ship, and what we can dream about making possible on these web surfaces." ^[raw/transcripts/lenny/guillermo-rauch.md]

**v0 inverts Git — start with intent, get code** — Unlike traditional engineering where you code first then summarize intent in a commit message, v0 flips this: you state what you want, and the code is the output. "The Git commit is you go into the chat and say, 'Please change the color of this button. And when I click it, save this form to a database.' And so you're starting with the intent and the output is the code." ^[raw/transcripts/lenny/guillermo-rauch.md]

**AI becomes synonymous with software — stop talking about it** — Guillermo predicts a future where AI is not discussed as a separate technology category but is simply how software is built. "We need to stop talking about AI at some point. I just see a future where AI becomes synonymous with software. We build software and we use software to build software." ^[raw/transcripts/lenny/guillermo-rauch.md]
