# AI Prototyping for Product Managers

## Synthesis

1. **AI prototyping transforms how PMs build products, generate ideas, and operate.** Colin Matthews (former PM, now teaching the most popular AI prototyping course) provides a complete guide: which tools to use when, battle-tested prompts, real-world examples, and a step-by-step workflow. The core promise: turn Figma designs into working apps with a few clicks, or turn a PRD into a prototype in minutes.

2. **The AI prototyping tool ecosystem clusters around key use cases:**
   - **Bolt and Lovable:** Best for full-stack web apps from a single prompt — ideal for customer-facing prototypes and internal tools.
   - **Replit:** Browser-based IDE with AI assistance — good for collaborative prototyping and quick scripts.
   - **v0 by Vercel:** Specializes in UI component generation from text descriptions — great for front-end exploration.
   - **Cursor/Windsurf:** AI code editors for more technical prototyping with human-in-the-loop.
   - **Claude Code:** Terminal-based agent for building scripts, automations, and full projects.
   - **ChatGPT Projects:** Best for PM workflows (PRDs, analysis, communications) rather than building software.

3. **The key workflow: describe → generate → test → iterate.** The PM provides natural language descriptions of what they want, the AI tool generates a working version, the PM tests it with real users or stakeholders, and iterates through feedback. The bottleneck shifts from "can we build this?" to "should we build this?"

4. **Prototyping is not production — and that's the point.** The goal is speed and learning, not production-quality code. PMs should treat AI-generated prototypes as disposable experiments to validate assumptions, test with users, and communicate ideas. Once validated, the prototype guides the engineering team's actual implementation.

5. **Concrete use cases for PMs:**
   - Turn a PRD into a clickable prototype to test with users before engineering commits.
   - Convert Figma designs into working front-ends to validate interaction patterns.
   - Build internal tools (dashboards, admin panels, data viewers) without consuming engineering time.
   - Generate A/B test variations to experiment with different UX approaches.
   - Create demo videos and walkthroughs from working prototypes for stakeholder buy-in.

6. **Getting unstuck is a skill itself.** Colin provides specific troubleshooting strategies: how to debug when the output isn't what you expected, how to break complex requests into smaller prompts, and how to use the AI tool itself to help you refine your request.

## Key Disagreements

- **Prototype fidelity:** Whether low-fidelity prototypes (quick and dirty) or high-fidelity ones (production-quality) generate better learning.
- **PMs building vs. PMs specifying:** Debate over whether PMs should build prototypes themselves or use AI to better specify what engineers should build.
