# Vibe Coding Examples

## Synthesis

1. **Vibe coding is producing real, regularly-used tools, not toys.** Lenny surveyed his community and received over 1,000 responses of people sharing vibe-coded tools they use regularly. The response demonstrated that non-technical people are building genuinely useful applications — from apartment buzzer apps to personalized greeting card generators to workplace accomplishment trackers.

2. **The most common use cases automate personal life admin.** Many of the most popular vibe-coded tools solve small but persistent personal pain points: automating apartment deliveries, managing personal finances, tracking habits, curating playlists, and generating personalized content. These are tasks too narrow for commercial software but perfectly suited for a quick AI-built solution.

3. **Bolt, Lovable, Replit, Cursor, and ChatGPT are the primary tools.** The vibe coding ecosystem clusters around a few key platforms: Bolt and Lovable for full-stack web apps from prompts, Replit for browser-based development, Cursor for code-assisted development, and ChatGPT for quick scripts and automations.

4. **Vibe coding changes the economics of software creation.** When building an MVP goes from weeks to minutes, the entire product development cycle compresses. PMs can prototype ideas, test them with real users, and iterate before involving engineering. This shifts the bottleneck from "can we build it?" to "what should we build?"

5. **The barrier to entry is now almost zero.** People with zero coding experience are building functional tools. The key skill is no longer programming — it's clearly describing what you want, iterating on outputs, and having good product judgment about what's worth building.

## Key Disagreements

- **Quality ceiling:** Some argue vibe-coded apps hit a quality ceiling for complex features; others counter that the tools improve so fast this ceiling keeps rising.
- **"Real" engineering vs. vibe coding:** Debate over whether vibe-coded prototypes can graduate to production-grade software without engineer involvement.
